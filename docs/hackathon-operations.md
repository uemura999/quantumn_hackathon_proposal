# ハッカソン運営マニュアル

> 「Opening Urban Challenges with Q」当日運営の手順書 (主催・メンター向け)

電通 × NTT 西の社会実装ハッカソン本番で、参加者 (高校生) を 4 時間または 2 時間で完走させるための運営手順をまとめています。

---

## 0. 前提

- 推奨参加人数: 1 チーム 3-4 人 × 4-6 チーム = 12-24 人
- 機材: ノート PC × 1 / チーム (Chrome / Edge 推奨)、プロジェクター、Wi-Fi (帯域 50Mbps 以上)
- スタッフ: 司会 1、メンター 1 / 2 チーム、審査員 3-5
- 当日資料: ストーリーカード雛形 / プレゼン雛形 / 質問プロンプトカード (templates/ 配下)
- 環境: Node 20+ で `npm run dev` 起動済み、`pnpm test` グリーン、Colab Notebook を GitHub Public 化済み

### 1.1 4 時間版の鉄則

- Tutorial と Challenge の合間に休憩を必ず挟む (集中力の山が 45 分で切れる)
- ストーリー設計には「もし台風で…」「もし通学路なら…」と質問プロンプトを切り出す
- Qiskit セッションは Colab pip install が遅いことを開始時に告知 (2分待つ覚悟)
- プレゼンは「問題 / アプローチ / 結果」の 3 スライドテンプレ強制

## 3. メンター動きの詳細

### 3.1 Tutorial 中 (0:15-1:00)

| ステップ | 観察ポイント | 介入セリフ例 |
|---|---|---|
| Step 0 | 配送先 6 つを認識できているか | 「いまトラックがどこから出発する?」 |
| Step 1 | 720 通りに「無理だ」と感じてるか | 「もし配送先が 10 個なら何通り?」 |
| Step 2 | 全候補が同じ高さに見えているか | 「右の候補バー、まだ差がついてないよね」 |
| Step 3 | γ=2.8 で何が起きたか言語化できるか | 「強すぎたら何が起きた? 1 位が変わったね」 |
| Step 4 | β=0 で動かない不思議さを感じているか | 「γ だけだとなぜ動かない?」 |
| Step 5 | reps を増やすと尖るのを観察したか | 「3 回目のレイヤーで何が起きた?」 |
| Step 6 | Qiskit ボタンに気づいているか | 「次のステップでこれを Python でも見るよ」 |

### 3.2 Challenge 中 (1:15-2:00)

- 5 分ごとに各テーブルのスコアを暗黙的にチェック
- 15 分経ってもベストが更新されないチームには:
  - 「γ を 1.2 で β を 0.5 に試したことある?」
  - 「reps を 3 にしてみた?」
- 後半 15 分の「渋滞切替」を 1:45 で合図 → 朝ラッシュにすると別ルートが浮かぶ

### 3.3 Qiskit Colab (2:45-3:20)

- 2:45 ちょうどで全員 Colab を開かせる
- pip install 中の 2 分: Cell ③④の解説を口頭でする
- 3:05 までに全チーム実行完了を目標
- 3:05-3:20 は **JS との比較表** を見ながら「一致してる?」を全員で確認

## 4. トラブル対応 FAQ

| 症状 | 対応 |
|---|---|
| Colab `pip install` が 3 分以上かかる | 全員待つ。事前の 30 分前にスタッフ PC でウォームアップ済みなら 1 分弱 |
| Wi-Fi が落ちた | JS engine だけで進行継続。Qiskit Colab は当日後の宿題として案内 |
| Colab で qiskit インポートエラー | `pip install qiskit==1.2.4 qiskit-aer==0.15.1` をバージョン固定で再実行 |
| 3D シーンが描画されない | ブラウザを Chrome / Edge に切り替え。WebGL がサポートされてない端末では二次受け PC を提供 |
| クリップボード貼り付けが効かない | Colab Button の「スニペットを表示」を押し、テキストを手動コピー |
| ストーリーが浮かばないチーム | 質問プロンプトカードを 1 枚渡す。「最近ニュースで気になった災害は?」 |
| 時間オーバーしそう | タイマー音声合図を 30 秒前に鳴らす。テンプレ強制で個人裁量を絞る |
| Qiskit と JS 結果が完全一致しない | サンプリング誤差 (4096 shots) で ±2% 程度は正常。markdown で予告済み |

## 6. PR・写真の取り扱い

- 全テーブルを背景にした集合写真は最後に必ず撮る
- 各チームの**プレゼン中の画面**を写真に収めると Dentsu のリリース素材として強い
- 参加者の顔出し同意は受付時にフォームで取る (高校生は保護者同意も)
- 公開素材: ストーリーカード PDF + 最終スコアボード + 集合写真

## 7. イベント後 (1 週間以内)

- 各チームのアウトプット URL を主催 Notion に集約
- レポート記事を Dentsu PR チームに渡す (素材: 写真 + ストーリー + スコア)
- 参加者に修了証 PDF + 全 Notebook URL リストをメール
- 次回開催に向けた改善メモを `docs/hackathon-retrospective.md` に追記
